## Developer Checklist

### Code Complete

- [ ] All code necessary for the feature is written and checked into the version control system as it pertains to the project.
- [ ] All tests pass without errors

### Code Review

- [ ] The code has been peer reviewed.
- [ ] The code has been reviewed with the Tech Lead to ensure quality and adherence to coding standards.
- [ ] All the necessary feedback items are closed by the developer of the code.

### Free of Technical Debts

- [ ] There are no new technical debts like static errors / unacceptable warnings introduced in the code.
- [ ] Tech and Design Debt would be captured as a backlog story and reviewed in refinement.

### Testing

- [ ] Test coverage is >= 90%
- [ ] Unit testing: Individual units or components of the code have been tested in isolation and no open defects. This also means “Unit Testing” is automated and it’s not manual testing.
- [ ] Integration testing: The user story has been tested in combination with other related features or modules and no open defects.
- [ ] Security testing: Potential security vulnerabilities for the user story have been identified and addressed, as applicable.
- [ ] Performance testing: The story has been tested to ensure it meets the desired performance metrics, as applicable.

### Documentation

- [ ] All required documentation (technical, diagrams, deployment) complete according to the acceptance criteria.
- [ ] Architecture Design Records (ADRs) created in GitHub to document major decisions.
- [ ] Changesets documentation has been included in Pull Request

## Product Owner Checklist

- [ ] The user story has been validated against the defined Acceptance Criteria.
